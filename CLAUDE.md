# StayConnected

An iOS app that nudges you to reconnect with a small, rotating set of contacts
each day. SwiftUI + Core Data, iOS 18+.

## Build & Run

- Open `StayConnected.xcodeproj` in Xcode and run the `StayConnected` scheme.
- Deployment target: iOS 18.5. Bundle id: `com.anujpatel.StayConnected`.
- Tests use the **Swift Testing** framework (`import Testing`, `@Test`), not
  XCTest, in `StayConnectedTests/`. Run with Cmd-U or `xcodebuild test`.

## Architecture

Plain MVVM, grouped by role:

- **Views/** — SwiftUI screens. `AppShellView` hosts the four-tab `TabView`
  (`HomeView`, `PoolView`, `SummaryView`, `SettingsView`) and the onboarding
  cover. `HomeView` is the canonical "today" screen.
- **ViewModels/** — `TodayViewModel`, `PoolViewModel`, `SettingsViewModel`
  (all `@MainActor ObservableObject`). Views often create a throwaway
  `TodayViewModel(context:)` per action rather than holding one.
- **Models/** — Core Data entity helpers (`Person+Helpers`,
  `DailyPick+Helpers`, `AppSettings+Helpers`) and small utilities
  (`Date+StartOfDay`).
- **Services/** — `SelectionService` (the pick-ranking algorithm),
  `NotificationsService` (adaptive reminders + streak math), `PhoneLink`
  (tel:/sms: URL building).

Persistence is `PersistenceController.shared` (Core Data), injected via the
`managedObjectContext` environment.

### Core Data entities

- **Person** — a contact in the pool (`contactIdentifier`, `displayName`,
  `isInPool`, `isPinned`, `lastCalledAt`, `lastPickedAt`,
  `timesPickedThisMonth`, `cadence` (see `ContactCadence`), `snoozedUntil`,
  and a one-line `note`).
- **DailyPick** — today's chosen contact identifiers, keyed to local
  start-of-day (`Date.startOfDay`). Identifiers are stored as a newline-joined
  `String` (`contactIdentifiersRaw`) via the `contactIdentifierList` accessor —
  CloudKit-compatible and order-preserving.
- **ConnectionEvent** — one logged connection; deduped to at most one per
  contact per day by `TodayViewModel.markCalled`.
- **AppSettings** — `picksPerDay`, `minGapDays`, `remindersEnabled`,
  `reminderTime` (singleton via `AppSettings.fetchOrCreate`).

## Key behaviors

- **Daily picks** are generated by `SelectionService.pickToday`, ranking pinned
  → never-contacted → most overdue relative to their own cadence (`overdueRatio`
  = daysSinceCall / cadenceDays) → least-picked-this-month, with light randomness
  in the tail. Contacts within `minGapDays` of their last pick are demoted to a
  fallback pool; snoozed people are skipped entirely.
- **Cadence:** each person has a `ContactCadence` (Close ~2wk / Regular ~1mo /
  Occasional ~3mo), set from the Pool row's long-press menu.
- **Auto check-in:** tapping Call/Message logs the connection automatically
  (only when the link opens); `markCalled` dedups to one per contact per day.
- **Snooze:** `TodayViewModel.snoozePick` defers one person (`snoozedUntil`) and
  swaps in a replacement — the "not today" escape valve.
- **Calm by design:** the UI deliberately avoids pace/quota shaming (no "behind
  pace", no Warning-red progress). Keep new copy warm and forward-looking.
- **Auto-generate:** `HomeView` generates today's picks on the first open of a
  new day (tracked by the `lastAutoGeneratedDay` AppStorage key), runs
  `monthRolloverIfNeeded` (resets `timesPickedThisMonth`), and shows a
  pool-too-small warning.
- **Reset Today** deletes the `DailyPick` *and clears `lastPickedAt`* on those
  people so a follow-up Generate is genuinely fresh; it also marks the day so
  auto-generate won't immediately undo the reset.
- **Dialing** always goes through `PhoneLink`, which preserves a leading `+`
  for international numbers.
- **Contacts I/O** (phone numbers, avatar thumbnails) runs on detached
  background tasks; never synchronously on the main thread.

## Gotchas

- The Xcode project uses **manual file references** in `project.pbxproj` (no
  synchronized/file-system groups). Adding a new source file requires four
  entries (PBXBuildFile, PBXFileReference, group child, Sources build phase) —
  model them on an existing file (e.g. `SelectionService.swift`) and use unique
  24-hex-char IDs.
- `docs/` contains historical version-evolution guides; treat them as a record
  of how the app got here, not as current-state documentation.
